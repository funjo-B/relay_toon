# Relay Toon 목업 이미지 제작 가이드

목업(`mockup/index.html`)의 각 컷 그림을 AI 생성 이미지로 교체하기 위한 문서입니다.

**사용법**
1. 아래 "① 프로젝트 브리핑"을 클로드(또는 이미지 생성 AI)에 먼저 붙여넣는다.
2. "② 컷별 프롬프트"에서 원하는 컷의 프롬프트를 이어서 붙여넣고 이미지를 받는다.
3. 받은 이미지를 **표에 적힌 파일명**으로 `mockup/images/` 폴더에 저장한다.
4. 끝. 목업이 해당 파일을 자동으로 감지해 SVG 대신 이미지를 보여준다.

---

## ① 프로젝트 브리핑 (먼저 붙여넣기)

```
나는 "Relay Toon"이라는 서비스의 목업 프레젠테이션용 일러스트를 만들고 있어.

[서비스 소개]
Relay Toon은 AI가 웹툰의 세계관을 기억·관리하고, 여러 명의 사용자가 하나의
작품을 함께 만들어가는 협업형 릴레이 웹툰 플랫폼이야.
- 사용자가 "철수가 복도를 걷는다" 같은 한 줄만 입력하면 AI가 컷·연출·말풍선을 자동 생성
- 3컷 단위로 누구나 이어 그리는 공개 릴레이 (끝나지 않는 웹툰)
- 원작의 원하는 화에서 분기하는 브랜치(평행세계) 시스템
- AI Story Bible이 캐릭터·장소·떡밥·연표를 기억해 설정 일관성 유지
- 기여도 기반 수익 분배와 AI 토큰 이코노미

[그림 스타일 가이드 — 모든 이미지 공통]
- 스타일: 밝고 깔끔한 한국 웹툰 그림체, 굵은 라인, 생동감 있는 색감
- 분위기: 유쾌하고 약간 병맛, 친근함
- 구도: 가로 16:9 (1600×900)
- UI 화면이 필요한 컷은 다크 테마 앱 화면(배경 #0f1117, 포인트 컬러 보라 #7c5cff)
  위에 웹툰 캐릭터가 등장하는 반실사 목업 느낌으로
- 등장 캐릭터 통일: "철수" = 안경 쓴 평범한 남학생(교복), 매 컷 동일한 외형 유지
- 텍스트는 최소화 (자리만 잡고 실제 글자는 대충 뭉개도 됨)

이제 컷별 프롬프트를 하나씩 줄게. 위 스타일을 유지해서 만들어줘.
```

## 참고

- 이미지가 없는 컷은 지금처럼 SVG 스케치가 그대로 표시됩니다 (섞여 있어도 무방).
- 전부 교체할 필요 없이 발표에 쓸 핵심 컷부터 만들면 됩니다. 추천 우선순위: `feat-main-1~4` → `intro-1` → `feat-open-1,2` → `feat-branch-3`.
- 파일 형식은 png 기준이지만 jpg를 쓰려면 `index.html`의 `imgPath` 확장자만 바꾸면 됩니다.
